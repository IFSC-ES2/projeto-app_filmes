# ADR 001 — Adoção de Arquitetura Desacoplada (React + Spring Boot)

* **Data:** 14 de maio de 2026
* **Status:** Aprovado
* **Responsáveis:** Isabella, Gabriel e Marcus

## 1. Contexto

No planejamento inicial da Sprint 1, o projeto CineLog foi concebido utilizando uma estrutura monolítica simples com páginas HTML estáticas servidas diretamente pelo Spring Boot. Com a necessidade de evoluir o MVP para uma interface de usuário mais dinâmica, com manipulação de estados em tempo real (como a listagem dinâmica de filmes e renderização condicional de notas), a estrutura antiga se mostrou limitada e de difícil manutenção.

## 2. Decisão

Decidimos quebrar o monólito e adotar uma arquitetura cliente-servidor totalmente desacoplada. O frontend foi migrado para uma Single Page Application (SPA) utilizando **React**, enquanto o backend em **Spring Boot** passou a funcionar estritamente como uma API RESTful, comunicando-se exclusivamente através do formato JSON.

## 3. Consequências (Benefícios e Trade-offs)

### Benefícios:

* **Separação de Responsabilidades:** O nosso dev pôde trabalhar de forma isolada na interface (React) e nas regras de negócio/persitência (Spring Boot).
* **Experiência do Usuário (UX):** A navegação e a atualização das listas de filmes ocorrem sem a necessidade de recarregar a página inteira no navegador.

### Trade-offs / Desvantagens:

* **Complexidade de Infraestrutura Local:** Agora é necessário gerenciar e iniciar dois servidores de desenvolvimento distintos para rodar o projeto localmente.
* **Problema de CORS:** Introduziu a necessidade de configurar explicitamente as permissões de Cross-Origin Resource Sharing (CORS) no backend para liberar as requisições vindas da porta do React.
