# ADR 002 — Uso do Banco de Dados H2 em Memória para o MVP

* **Data:** 14 de maio de 2026
* **Status:** Aprovado
* **Responsáveis:** Isabella, Gabriel e Marcus

## 1. Contexto

Para validar as funcionalidades essenciais do CineLog (cadastrar títulos, salvar notas e listar a biblioteca pessoal), o sistema exige um banco de dados relacional estável. No entanto, configurar instâncias locais de bancos de dados tradicionais (como PostgreSQL ou MySQL) exigiria scripts complexos de instalação e configuração por parte de cada integrante do time e do professor no momento da correção.

## 2. Decisão

Decidimos utilizar o banco de dados **H2 Database rodando em modo em memória**, gerenciado de forma automática através do Spring Data JPA.

## 3. Consequências

### Benefícios:

* **Configuração Zero (Zero-Setup):** O banco é inicializado junto com a aplicação Spring Boot, facilitando a execução local do projeto com um único comando.
* **Agilidade nos Testes:** Ideal para a esteira de Integração Contínua (CI) no GitHub Actions, que consegue rodar os testes unitários e de integração limpando e recriando as tabelas instantaneamente a cada execução.

### Trade-offs / Desvantagens:

* **Volatilidade dos Dados:** Como o banco roda estritamente na memória RAM, todos os registros de filmes e usuários cadastrados são perdidos sempre que o servidor do backend é reiniciado ou desligado. Essa limitação foi aceita pelo time por ser perfeitamente tolerável na fase de validação de um MVP.
